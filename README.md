# 札幌 自費リハビリナビ（リデザイン版）

理学療法士・作業療法士・言語聴覚士など身体の専門職が関わる、札幌市内の
自費リハビリ・整体・ピラティス・ジム・運動支援施設を探せる地域ポータルサイトです。

Next.js 14（App Router）+ TypeScript + Tailwind CSS

## セットアップ

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # 本番ビルド（Vercelと同じ）
```

既存のVercelプロジェクトに反映する場合は、このフォルダの中身でリポジトリを
置き換えて push してください。

## 施設情報の管理（data/facilities.ts）

施設の追加・編集・公開切り替えは `data/facilities.ts` だけで完結します。

| フィールド | 説明 |
|---|---|
| `isPublic` | `true` = 公開画面に表示 / `false` = 非表示（データは保持） |
| `status` | `"official"` 正式掲載（フルカード） / `"candidate"` 掲載候補（確認中リスト） / `"checking"` 情報確認中（件数のみ表示） |
| `adminMemo` / `checkedAt` / `priority` | 運営用メモ。**公開画面には表示されません** |

- 正式掲載に切り替えるとき：掲載許可・資格者在籍・料金・住所を施設に確認のうえ、
  `status: "official"` に変更してください。
- 新しい施設を追加するとき：配列にオブジェクトを1つ追加するだけで、
  一覧・件数・各フィルターに自動反映されます。
- カテゴリ・目的のマスタも同ファイル内（`CATEGORIES` / `PURPOSES`）にあります。

## 無料掲載の申し込みリンク

`components/RecruitSection.tsx` 内のコメント部分を、Googleフォームまたは
メールアドレスに差し替えてください。

## 構成

```
app/            レイアウト（SEOメタ・フォント）とトップページ
components/     セクションごとのコンポーネント
data/           施設データ・カテゴリ・目的・エリアのマスタ
```

将来、施設詳細ページを作る場合は `app/facilities/[id]/page.tsx` を追加し、
`data/facilities.ts` の `id` をそのままURLに使えます。

## 表現ガイドライン

医療広告と誤解される表現（治療します／診断します／必ず改善します 等）は
使用しません。「身体機能の維持・改善を支援」「運動継続を支援」
「医療保険・介護保険外の自費サービス」などの表現に統一しています。
